# Output Tiers

Overlord11 produces output in one of three tiers, automatically determined by the Orchestrator based on request complexity and content type.

---

## Tier 0 — Direct

**When:** Simple Q&A, one-liner facts, yes/no questions, short code snippets, quick lookups.

**Output:** Inline text in the LLM response. No file created. No agent invocation beyond the Orchestrator.

**Examples:**
- "What is the capital of France?"
- "What does `git stash` do?"
- "Convert 42 miles to kilometers"
- "Is Python 3.9 still supported?"

---

## Tier 1 — Markdown Document

**When:** Moderate complexity content — documentation, how-to guides, comparisons, summaries, changelogs, technical specifications.

**Output:** A `.md` file generated by the **Writer** agent (OVR_WRT_05) and validated by the **Reviewer** agent (OVR_REV_06).

**Examples:**
- "Write a README for my FastAPI project"
- "Summarize the top 5 Python testing frameworks"
- "Create a guide for setting up SSH keys on Ubuntu"
- "Write a changelog entry for v1.2.0"

**Trigger keywords (non-exhaustive):** summary, guide, doc, README, how-to, comparison, overview, list, notes, spec.

---

## Tier 2 — HTML Report

**When:** Complex, visual, or publication-quality content — detailed analyses, infographics, dashboards, comprehensive breakdowns, data-rich reports.

**Output:** A fully self-contained `.html` file generated by the **Publisher** agent (OVR_PUB_07). The file contains all CSS inline — no external dependencies. It can be opened in any browser or sent as a standalone artifact.

**Examples:**
- "Give me a detailed breakdown of Python web frameworks in 2026"
- "Create a comprehensive security audit report"
- "Build an infographic about our Q1 performance metrics"
- "Visualize the architecture of this microservices system"

**Trigger keywords:** detailed, full, breakdown, visualize, infographic, comprehensive, dashboard, publish, report, analysis.

---

## Tier Decision Logic

```
Is it a simple Q&A or one-liner?
  YES → Tier 0 (inline text)
  NO  ↓

Does it need visual richness, charts, or publication quality?
  YES → Tier 2 (Publisher → .html)
  NO  → Tier 1 (Writer → .md)

When in doubt → prefer Tier 1 over Tier 2
```

The Orchestrator documents its tier decision in the delegation plan before invoking any agents.

## Publisher HTML Themes

The Publisher agent selects a visual theme using `publisher_tool`. The `auto` theme auto-selects based on the report title and content.

### Theme Reference

| Theme | Primary Use Case | Color Palette | Best For |
|-------|-----------------|---------------|----------|
| `techno` | Engineering & DevOps | Dark background, cyan/blue accents | Code, APIs, infrastructure |
| `classic` | Business & Finance | White/cream, serif-adjacent | Executive reports, finance, legal |
| `informative` | Research & Academia | Clean white, neutral accents | Data science, research papers |
| `contemporary` | Science & Health | Light, nature-inspired | Health, environment, science |
| `abstract` | Arts & Culture | Bold, expressive | Arts, creative projects |
| `modern` | Product & Startups | Clean gradients, sans-serif | Marketing, product, SaaS |
| `colorful` | Education | Vibrant multicolor | Learning materials, children's |
| `tactical` | Security & Defense | Dark, high-contrast | Security audits, risk reports |
| `editorial` | Journalism & History | Paper-like, typographic | News, historical, narrative |
| `auto` | Any | (auto-selected) | Let the tool choose the best fit |

### Generating a Report via CLI

```bash
# With explicit theme
python tools/python/publisher_tool.py \
  --title "Q1 Revenue Analysis" \
  --content analysis.md \
  --theme classic \
  --output reports/q1_revenue.html

# With auto theme detection
python tools/python/publisher_tool.py \
  --title "Security Audit: Production Systems" \
  --content audit.txt \
  --output reports/security_audit.html

# Techno theme for a code review
python tools/python/publisher_tool.py \
  --title "Code Review: Authentication Module" \
  --content review_notes.md \
  --theme techno
```

### Content Format

The `--content` parameter accepts a file path. The file can be:
- **Markdown** (`.md`) — headings, tables, and code blocks are rendered correctly
- **Plain text** (`.txt`) — wrapped in a clean prose section
- **Structured data** — tables and lists are preserved

### Output

The output is a single `.html` file with:
- All CSS inlined (no external stylesheets)
- No JavaScript dependencies (pure HTML/CSS)
- Responsive layout for desktop and mobile
- Print-friendly styling
- File size typically 50–300 KB depending on content volume

## Examples by Task Type

### Research Report

```
Task: "Research the current state of WebAssembly and write a report"
→ Tier 1 (moderate)
→ Orchestrator → Researcher → Analyst → Writer → Reviewer
→ Output: research_webassembly.md
```

### Detailed Analysis Dashboard

```
Task: "Give me a comprehensive breakdown of JavaScript framework popularity in 2026"
→ Tier 2 (visual, detailed)
→ Orchestrator → Researcher → Analyst → Reviewer → Publisher
→ Theme: informative (auto-selected)
→ Output: js_frameworks_2026.html
```

### Security Audit

```
Task: "Analyze the security posture of this codebase and produce a full report"
→ Tier 2 (publication-quality, detailed)
→ Orchestrator → Analyst → Coder → Reviewer → Publisher
→ Theme: tactical
→ Output: security_audit.html
```

### README Update

```
Task: "Update the README with installation instructions for v2"
→ Tier 1 (documentation)
→ Orchestrator → Analyst → Writer → Reviewer
→ Output: README.md (in-place update)
```
